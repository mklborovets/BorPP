ІНСТРУКЦІЯ ЗАПУСКУ АУКЦІОННОЇ СИСТЕМИ
==============================

1. ВИМОГИ ДО СИСТЕМИ
--------------------
- Java 17 або новіша
- PostgreSQL 12 або новіша
- Доступ до інтернету (якщо програма використовує зовнішні сервіси)

2. НАЛАШТУВАННЯ БАЗИ ДАНИХ
--------------------------
1) Встановіть PostgreSQL, якщо він ще не встановлений
   Завантажити можна з: https://www.postgresql.org/download/

2) Створіть базу даних:
   - Відкрийте pgAdmin або інший інструмент для роботи з PostgreSQL
   - Створіть нову базу даних з назвою "auction_db"
   - Відкрийте інструмент запиту SQL (Query Tool)
   - Імпортуйте дамп бази даних з файлу database_backup.sql

   АБО через командний рядок:
   - psql -U postgres -c "CREATE DATABASE auction_db;"
   - psql -U postgres -d auction_db -f database_backup.sql

3. НАЛАШТУВАННЯ ПІДКЛЮЧЕННЯ ДО БАЗИ ДАНИХ
----------------------------------------
1) Відкрийте файл database.properties у текстовому редакторі
2) Змініть налаштування підключення, якщо потрібно:
   - db.url=jdbc:postgresql://localhost:5432/auction_db
   - db.username=postgres
   - db.password=14256879
   - db.driver=org.postgresql.Driver

4. ЗАПУСК ПРОГРАМИ
----------------
Метод 1: Запуск JAR-файлу
- Переконайтеся, що Java встановлена і доступна в системі
- Двічі клацніть на файлі auction-system-1.0-SNAPSHOT.jar
  АБО запустіть через командний рядок:
  java -jar auction-system-1.0-SNAPSHOT.jar

Метод 2: Збірка і запуск з вихідного коду
- Встановіть Maven, якщо він ще не встановлений
- Відкрийте командний рядок у директорії проекту
- Виконайте команду: mvn clean package
- Запустіть згенерований JAR-файл: java -jar target/auction-system-1.0-SNAPSHOT.jar

5. ВИРІШЕННЯ ПРОБЛЕМ
------------------
- Якщо програма не запускається, перевірте наявність Java 17+
- Якщо виникають проблеми з підключенням до бази даних:
  - Перевірте, чи запущений сервер PostgreSQL
  - Перевірте налаштування в database.properties
  - Перевірте, чи правильно створена база даних і імпортований дамп
